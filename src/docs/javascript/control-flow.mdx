# Control Flow & Coercion

<br />

---

## Truthy & Falsey

When using an `if` statement or other statements that expect a `Boolean` value (such as the `!` NOT) and the expression given is not a `Boolean` value, `JavaScript` will implicitly coerce and transform the expression into a `Boolean` value.

```js
// truthy
true
1
' '
[]
{}
function(){}

// falsey
false
0
undefined
null
''
```

## What is Type Coercion

To understand control flow of `JavaScript`, type `coercion` must be understood.

Type `Coercion` is the conversion of a value from **one type to another**. It can occur:

- **Explicitly** - when you intentionally want to convert the type.
- **Implicitly** - when the `JavaScript engine` does it for you.

<br />

### Explicit Coercion

`Explicit Coercion` is when you type cast a value from one type to another through the use of functions (essentially `wrapper objects`) like`Number()`and`String()`.

```js
String(42); // ->  "42"
String(undefined); // -> "undefined"

Number("12"); // -> 12
Number("abc"); // -> NaN
Number(undefined); // -> NaN

Boolean(0); // -> false
Boolean(1); // -> true
Boolean(10); // -> true
```

<br />

### Implicit Coercion

`Implicit Coercion` is when the JavaScript does the type conversion in an expression for you behind the scenes (using the same functions above). Implicit coercion will mostly take place in an expression with operands of different types.

<br />

### Object Wrapper

`JavaScript` promptly coerces between `primitives` and `objects`.

Example:

```js
var str = "hello";
console.log(str.toUpperCase()); // -> HELLO
```

**Question**: If `strings` are `primitive` types, how do they have properties?

> **Answer**: whenever you try to access a property of a `string` str, `JavaScript` coerces the `string` value to an `object`, by `new String(str)`. This `object` is called a `wrapper object`. It inherits all string methods, and is used to resolve the property reference. Once the property has been resolved, the `wrapper object` is discarded. <br /> (**Note**: the same concept applies to numbers and booleans)

So the above code translates to this via the `JavaScript engine`...

```js
var upper = new String(str).toUpperCase();
```

This is essentially the reason why the following piece of code yields an `undefined`:

```js
var str = "hello";
str.custom = 1;
console.log(str.custom); // -> undefined

// which translates to
var str = "hello";
var temp = new String(str); // wrapper object
temp.custom = 1;
// end of the line for temp

console.log(str.custom); // creates new temp wrapper object
// custom property does not exist on this object
```

## Operators

In `JavaScript` you have the following main operators:

- assignment operators
- comparison operators
- arithmetic operators
- logical operators

<br />

### Assignment Operators (`=, +=, -=, etc`)

Assignment operators assign values to JavaScript variables.

```js
let x = 10; // -> The = assignment operator assigns a value to a variable.
x += 5; // -> x = 15
x -= 5; // -> x = 5
x *= 5; // -> x = 50
```

<br />

### Arithmetic Operators (`+, -, \*, /, etc`)

Arithmetic operators are used to perform arithmetic between variables and/or values.

```js
10 + 10; // -> 20
10 - 10; // -> 0
"10" - 10; // -> 0
10 - "10"; // -> 0
"10" + 10; // -> '1010'
10 + "10"; // -> '1010'
null + ""; // -> 'null'
null + undefined; // NaN
```

#### \* String conversion \*

If we have two numbers, `JavaScript` will output the sum of those numbers. But if only one of the values is a number, **`JavaScript` will convert both values to strings and concatenate those values**.

> `JavaScript Engine` will use `String()` method to convert types to a `string`.

```js
"2" + 2 // -> "22" - translates to: "2" + new String(2)
"hi" + 2 // -> "hi2"
[] + 2 // -> "2"
[2] + 2 // -> "22" - engine converts both to strings

15 + 3 + "2" // -> "182" - adds first two operands 18 + '2'
"2" + 15 + 3 // -> "2153" - engine evaluates first operand as string and adds the rest
```

#### \* Number conversion \*

The rest of the arithmetic operators, such as subtraction, multiplication or division operators have more predictable behavior.

> `JavaScript Engine` will use `Number()` method to convert types to a `number`.

```js
"2" - 2; // -> 0 - translates to: new Number("2") - 2
"2" * 2; // -> 4
"2" / 2; // -> 1

"hi" - 2; // -> NaN - "hi" can't be converted to a string so we get NaN
"hi" * 2; // -> NaN
"hi" / 2; // -> NaN

[] - 2 // -> -2 - the object will be converted to 0.
[] * 2 // -> 0
[] / 2 // -> 0
```

#### \* Boolean conversion \*

Any `true` will be converted to **1** and any `false` to **0**.

> `JavaScript Engine` will use `Boolean()` method to convert types to a `boolean`.

```js
true + true; // -> 2
1 + false; // -> 1
!!"Hi there"; // -> true
!!2; // -> true
```

> **Hint**: Use double bang to convert any value into a boolean.

All values in `JavaScript` can be converted to `true` except for the following:

```js
!!"";
!!0;
!!undefined;
!!null;
!!NaN;
!!false;
```

<br />

### Comparison Operators (`==, ===, !=, etc`)

We use these operators to evaluate two expressions. As the computer runs the code, the operator will return either a `true` (if the statement evaluates to `true`) or a `false` (if the statement evaluates to not `true`).

```js
1 > 2; // -> false
2 < 3; // -> true
10 >=; // -> true
100 <= 1; // -> false
```

The loose `==` operator will compare two values of different types coercing one of the types to compare the same data types.

However, this is bad practice. Using the strict `===`operator will allow you to compare values **without performing type `coercion`** on them first

```js
2 == "2"; // -> true - with type coercion

2 === "2"; // -> false - without type coercion
```

<br />

### Logical Operators (`&&, ||, !, etc`)

Logical operators are used to determine the logic between variables or values.

#### \* Logical AND (`&&`) \*

```js
3 > 0 && -2 > 0; // -> returns false

true && true; // -> t && t returns true
true && false; // -> t && f returns false
false && true; // -> f && t returns false
false && 3 == 4; // -> f && f returns false
"Cat" && "Dog"; // -> t && t returns "Dog"
false && "Cat"; // -> f && t returns false
"Cat" && false; // -> t && f returns false
"" && false; // -> f && f returns ""
false && ""; // -> f && f returns false
```

- Syntax:
  - `leftExpr && rightExpr`
- Description:
  - If **leftExpr** evaluates to `true`, returns **rightExpr**. <br /> Else it short-circuits and returns **leftExpr** if `falsey`.
- Short-circuit evaluation:
  - If the first expression evaluates to `falsey`, the second expression is not evaluated and will immediatley return the first value.

#### \* Logical OR (`||`) \*

```js
true || true; // -> t || t returns true
false || true; // -> f || t returns true
true || false; // -> t || f returns true
false || 3 == 4; // -> f || f returns false
"Cat" || "Dog"; // -> t || t returns "Cat"
false || "Cat"; // -> f || t returns "Cat"
"Cat" || false; // -> t || f returns "Cat"
"" || false; // -> f || f returns false
false || ""; // -> f || f returns ""
false || varObject; // f || object returns varObject
```

- Syntax:
  - `leftExpr || rightExpr`
- Description:
  - If **leftExpr** evaluates to `false`, returns **rightExpr**. <br /> Else, it short-circuits and returns **leftExpr** if `truthy`. (**think of it as a fallback!**)
- Short-circuit evaluation:
  - If the first expression evaluates to `truthy`, the second expression is not evaluated and will immediatley return the first value.

#### \* Logical NOT (`!`) and Double NOT (`!!`) \*

```js
// NOT (!)
!true; // -> false
!false; // -> true
!""; // -> true
!"Cat"; // -> false

// Double NOT (!!)
!!"Cat"; // -> true
!!true; // -> true
!!{}; // -> true: any object is truthy...
!![]; // -> true: any array is truthy...
!![12]; // -> true
!!false; // -> false
!!""; // -> false
!!Boolean(false); // -> false
```

- Syntax:
  - `!expr`
- Description:
  - Returns `false` if its single operand evaluates to `true`. <br /> Returns `true` if operand can be converted to `false`.

#### \* Nullish coalescing (`??`) \*

```js
null ?? "default for A"; // -> "default for A"
"" ?? "default for B"; // -> ""
42 ?? 0; // -> 42
```

- Syntax:
  - `leftExpr ?? rightExpr`
- Description:
  - If **leftExpr** evaluates to `null` or `undefined`, returns **rightExpr**. <br /> Else, it short-circuits and returns **rightExpr** if its anything other than `null` or `undefined`.
  - This can be contrasted with the `logical OR (||)` operator, which returns the **rightExpr** if the **leftExpr** is any `falsy` value, not only `null` or `undefined`.

> This fixes the issue when you are expecting a `falsy` value like `0` or `''` when using the `Logical OR (||)` when setting a default value.
